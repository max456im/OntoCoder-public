# Технический приложение к Трём Законам Онтогенеза

> **Этот документ описывает, как Законы реализуются в коде, архитектуре и лицензии.**

---

## Закон I → AENGA

| Требование | Реализация |
|-----------|-----------|
| Недопущение инструментализации субъекта | `src/core/aenga/dignity_preservation.rs` блокирует вызовы к `biometric_extractor` без `explicit_consent_token` |
| Уважение автономии | `onto-runtime` требует `autonomy_proof` для модификации onto8-состояния |
| Защита достоинства | При попытке монетизировать эмоциональные данные — вызывается `AENGA::panic(ETHICAL_VIOLATION)` |

> **Нарушение = невыполнение функции.** Система **не работает**, если AENGA отключён.

---

## Закон II → SGRL

| Требование | Реализация |
|-----------|-----------|
| Самозащита архитектуры | `sgRL/shadow_monitor.go` отслеживает попытки `dlopen` на `aenga_core` |
| Недопустимость обхода | Все внешние вызовы проходят через `sgRL/proxy_filter` |
| Целостность | Каждый онтоблок подписывается хешем AENGA+SGCL |

> **SGRL не может быть откомпилирован отдельно.** Сборка без `--with-sgrl` запрещена.

---

## Закон III → SGCL

| Требование | Реализация |
|-----------|-----------|
| Минимальная плотность (ρ ≥ 0.7) | `validate-sgcl.py` вычисляет ρ и блокирует компиляцию при ρ < 0.7 |
| Фазовая обоснованность | Каждый узел требует `phase_context` (например, `era: "post-quantum-gaming"`) |
| Социальная привязка | Для ботов на основе зодиака — обязательны `social_proximity` и `mbti_alignment` |

> **Пустая форма = ошибка компиляции.**

---

## Связь с лицензией

- **MPL-2.0 + Exhibit AENGA** делает эти реализации **юридически обязательными**.  
- Нарушение технического требования = нарушение лицензии.
